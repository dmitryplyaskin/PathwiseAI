# Главная страница (Dashboard) — приоритеты обучения

## Цель

Главная страница (`/`) должна снимать когнитивную нагрузку и отвечать на вопрос: **что делать прямо сейчас**.

Порядок блоков соответствует ежедневному сценарию пользователя:

1. **Повторить**
2. **Слабые места**
3. **Не изучал**
4. **Открыть новое**

## Текущее поведение (Frontend)

Файл: `frontend/src/pages/home/ui/HomePage.tsx`

### 1) Повторить

- Источник данных: `GET /lessons/for-review/:userId` (SM-2).
- Отображение:
  - счётчики: **Просрочено / Сегодня / Скоро**
  - список top-N уроков для повторения
  - кнопки: **Начать** (открывает первый урок из очереди), **Все** (ведёт на `/review`)

### 2) Слабые места

- Источник данных: `GET /exams/user/:userId` (история тестов).
- Логика (на текущем этапе):
  - экзамены сопоставляются с уроками через заголовок экзамена `Тест по уроку: {название урока}` и `courseId`
  - “слабое место” = низкий средний балл по последним попыткам (порог по умолчанию: < 70%)

> Примечание: сейчас сущность `Exam` на бэкенде не хранит прямую ссылку на урок. Это улучшение можно сделать позже, добавив `lessonId` в модель/ответ.

### 3) Не изучал

- Источник данных: `GET /lessons`
- Уроки отбираются по статусу `not_started` и отображаются top-N карточками.

### 4) Открыть новое

- Действия открывают модальное окно создания контента:
  - “Создать урок”
  - “Создать курс”

## Всегда видимая кнопка создания

На странице присутствует фиксированная кнопка **“Создать”** (FAB), которая всегда доступна при прокрутке и открывает создание контента.
