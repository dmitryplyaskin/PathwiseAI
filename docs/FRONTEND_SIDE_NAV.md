# Frontend: боковое меню (SideNav)

## Зачем

Добавлено левое боковое меню (как в типичном SaaS), чтобы упростить навигацию по приложению:

- на мобилке открывается по плавающей кнопке “бургер” (хедер не используется)
- содержит ссылки на основные страницы (роуты)
- блок профиля и выход — закреплены внизу меню
- ссылка `PathwiseAI` на главную из хедера убрана (название осталось текстом)
- на **десктопе** меню отображается **постоянно (permanent)**, а на **мобилке** — как **выезжающее (temporary)**
- на **десктопе** меню можно **свернуть в узкую полосу (mini)**; в свернутом состоянии для пунктов используются **tooltip**

## Где реализовано

- `frontend/src/app/ui/Layout.tsx`
  - показывает плавающую кнопку меню на мобилке
  - подключает `SideNav` и управляет состоянием `open/close`
  - управляет состоянием mini-режима (collapsed) на десктопе
- `frontend/src/widgets/side-nav/ui/SideNav.tsx`
  - MUI `Drawer` (temporary)
  - список навигации `NAV_ITEMS`
  - нижний блок: профиль (`/profile`) + выход (`logoutRequested`)
  - принудительно отключено скругление `Drawer`-Paper (иначе применяется глобальный `MuiPaper.borderRadius`)
  - поддержан mini-режим (collapsed) + кнопка сворачивания/разворачивания

## Какие роуты используются

Фактическая конфигурация роутов находится в:

- `frontend/src/app/config/router.tsx`

В меню сейчас выведены основные “статические” маршруты, которые не требуют параметров:

- `/` (Главная)
- `/courses` (Курсы)
- `/test-history` (История тестов)
- `/review` (Повторение)
- `/profile` (Профиль, внизу)

Динамические страницы вида `/courses/:id/...` открываются через сценарий навигации из списка курсов/юнитов/уроков.

## Как добавить новый пункт меню

Добавьте элемент в `NAV_ITEMS` в `frontend/src/widgets/side-nav/ui/SideNav.tsx`:

- `label` — подпись
- `to` — путь
- `icon` — иконка MUI
- `match` — `exact` или `prefix` для подсветки активного пункта
